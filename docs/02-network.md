# 网络规划
- 目标：做到“隔离 + 可控访问 + 可排障”
- 当前物理条件：单网口 / 无 VLAN / 单广播域 192.168.5.0/24  
- 实现方式：逻辑分区 + 固定 IP + 访问控制收敛

## 网段规划（逻辑分区）
> ⚠️ 说明：由于当前没有 VLAN，本质仍是同一 /24 网段。下列为“逻辑隔离区段”，通过 IP 规划 + 防火墙实现访问控制。

- **mgmt:** `192.168.5.1–19`（PVE 管理、仅管理终端可达）
  - 路由器：192.168.5.1
  - PVE：192.168.5.10
  - 管理电脑：192.168.5.11
- **tools:** `192.168.5.20–39`（日常工具 LXC/VM，如反代、监控、DNS、日志）
- **lab:** `192.168.5.40–69`（K8s 学习区 / 实验环境）
- **ai:** `192.168.5.70–79`（OpenClaw 等高风险区 / 下载类 / 可疑代码区）
- **clients (DHCP):** `192.168.5.100–199`（手机 / 笔记本 / 家庭终端）
- **reserve:** `192.168.5.80–99 / 200–254`（未来扩展 / IoT / Guest）

## 访问控制原则
> 本质：管理面收敛 + 高风险区限制横向移动

### 核心原则
- `ai -> mgmt`：默认禁止  
- `ai -> lab`：仅允许访问 K8s API / 必要服务端口（最小化）  
- `tools -> lab`：允许（用于日常访问监控 / 面板等）  
- `lab -> mgmt`：默认禁止  
- `clients -> mgmt`：仅允许 192.168.1.11  
- 外部访问：优先 VPN / Zero Trust；禁止直接暴露 PVE 管理界面

### 管理面收敛规则（关键）
仅允许：
```
192.168.5.11 → 192.168.5.10 TCP 8006
192.168.5.11 → 192.168.5.10 TCP 22
```

拒绝：
```
ANY → 192.168.1.10 TCP 8006
ANY → 192.168.1.10 TCP 22
```

## DNS / 域名（示例）
> 提高可读性与排障效率

- 内网域：
  - `*.lab.home`
  - `*.tools.home`
  - `*.ai.home`

- 示例映射：
  - `proxy.tools.home → 192.168.5.20`
  - `k8s.lab.home → 192.168.5.40`
  - `llm.ai.home → 192.168.5.70`

- 统一反代入口：
  - `proxy.tools.home`

## 设计逻辑说明
当前环境无法做二层隔离，因此采用：

- 单广播域
- 逻辑分区
- 管理面收敛
- 风险区限制横向访问

实现效果：

- 管理平面不暴露
- 高风险工作区不可横向攻击基础设施
- IP 一眼可识别角色
- 三年内无需重构
